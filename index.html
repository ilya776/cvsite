About Me

Illia Bilous

DevOps Engineer

I specialize in designing, implementing, and maintaining CI/CD pipelines using GitLab CI and Jenkins for multiple client projects. My expertise includes managing containerized applications using Docker and Kubernetes, including writing custom Helm charts and configuring Ingress Controllers.

I have extensive experience provisioning and supporting cloud infrastructure on AWS (EC2, EKS, RDS, S3) and DigitalOcean, as well as integrating Vault for secure secret management within deployment pipelines and Kubernetes workloads.

My responsibilities include setting up QA and staging environments, automating deployment processes, and troubleshooting deployment issues. I monitor applications using Prometheus and Grafana, configuring alerting and dashboards for production systems.

I work with MySQL and Redis, including backup strategies, performance tuning, and recovery operations. I support application lifecycle operations such as rolling updates, rollbacks, versioning, and release coordination.

I collaborate with developers and QA to improve release workflows and streamline feedback loops, and write Bash scripts to automate repetitive DevOps tasks, provisioning steps, and system checks.

Automated CI/CD Pipeline
Project Overview

Designed and implemented a fully automated CI/CD pipeline using GitLab CI for a microservices architecture, reducing deployment time by 70% and enabling multiple daily deployments.

Challenges

The main challenge was integrating multiple microservices with different technology stacks into a unified deployment pipeline while ensuring zero-downtime deployments.

Solutions
  • Implemented a multi-stage pipeline with parallel execution for faster builds
  • Created custom Docker images for each microservice to standardize the build environment
  • Utilized Kubernetes for container orchestration with rolling update strategy
  • Integrated automated testing at multiple stages of the pipeline
  • Implemented automatic rollback mechanisms for failed deployments
Technologies Used

GitLab CI Docker Kubernetes Helm AWS EKS Bash

Results

The implementation resulted in a 70% reduction in deployment time, increased deployment frequency from weekly to daily, and reduced deployment-related incidents by 85%.

Cloud Infrastructure Automation
Project Overview

Created a scalable cloud infrastructure on AWS using Infrastructure as Code principles with automated monitoring and alerting, enabling rapid scaling during peak traffic periods.

Challenges

The main challenge was designing an infrastructure that could automatically scale to handle unpredictable traffic spikes while maintaining cost efficiency during low-traffic periods.

Solutions
  • Implemented infrastructure as code using Terraform for reproducible environments

  • Set up automated alerting for potential issues before they impact users
Technologies Used

AWS Terraform Prometheus Grafana CloudWatch Lambda

Results

The infrastructure successfully handled a 300% increase in traffic during promotional events with no performance degradation, while reducing overall cloud costs by 25% through optimized resource allocation.
